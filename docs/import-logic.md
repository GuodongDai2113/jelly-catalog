# 产品导入逻辑

本文档说明 `Jelly Catalog` 当前后台导入流程的执行顺序、字段约定与分类处理规则，便于后续维护导入模板和排查问题。

## 入口与文件

导入页面支持上传 3 类文件：

- `CSV File`：产品 CSV。
- `Category CSV File`：产品分类 CSV。
- `Images ZIP File`：产品图片压缩包，可选。

限制规则：

- 产品 CSV 和分类 CSV 至少上传一个。
- 如果同时上传产品 CSV 和分类 CSV，会先导入分类，再导入产品。
- 如果只上传分类 CSV，则只执行分类导入，不创建产品导入任务。
- 图片 ZIP 只在导入产品时使用；只导入分类时不会处理图片。

## 总体流程

### 1. 创建导入任务前

系统会先执行以下步骤：

1. 清理旧的临时目录。
2. 清空旧日志文件。
3. 记录本次导入开始日志。
4. 检查是否至少上传了一个 CSV 文件。

### 2. 分类优先导入

如果上传了分类 CSV：

1. 将分类 CSV 保存到临时目录。
2. 逐行读取分类 CSV。
3. 按 `slug` 和父级关系查找已有分类。
4. 已存在则更新，不存在则创建。
5. 处理分类描述与 Rank Math 字段。

如果没有上传产品 CSV，则分类导入完成后直接结束。

### 3. 产品导入任务创建

如果上传了产品 CSV：

1. 将产品 CSV 保存到临时目录。
2. 如果上传了图片 ZIP，则解压到临时目录。
3. 扫描 CSV 表头，统计 FAQ 列、属性列、产品总行数。
4. 建立图片索引，减少导入时的重复递归查找。
5. 保存导入任务状态，进入 AJAX 分批处理。

### 4. 产品分批导入

后台通过 AJAX 按批次处理产品行，支持：

- 断点续传
- 导入锁
- 单行失败重试
- 导入完成后清理临时目录

## 分类 CSV 结构

分类 CSV 当前支持以下字段：

- `Title`
- `Slug`
- `Parent Slug`
- `Description`
- `rank_math_focus_keyword`
- `rank_math_title`
- `rank_math_description`

说明：

- `Title` 与 `Slug` 至少需要有一个有效值。
- `Parent Slug` 用于建立父子层级。
- 如果 Rank Math 插件启用，对应 SEO 字段会一并导入。
- 已存在的分类会优先更新，而不是重复创建。

## 产品 CSV 识别与更新规则

### 产品匹配优先级

导入产品时，系统按以下顺序尝试匹配已有产品：

1. `ID`
2. 断点记录中的已创建产品 ID
3. `Slug`
4. 未匹配到则创建新产品

### 产品基础字段

当前产品 CSV 主要支持以下关键字段：

- `ID`
- `Title`
- `Slug`
- `Content`
- `Excerpt`
- `Status`
- `product_sku`
- `Categories`
- `Category Slugs`

以及图片、FAQ、属性和 Rank Math 扩展字段。

说明：

- `Slug` 会写入产品 `post_name`。
- `Status` 支持导入为 `publish`、`draft` 等状态；草稿导出后再导入仍会保持草稿。
- `product_sku` 会写入产品 meta `product_sku`。
- Rank Math 启用时，可导入焦点关键词、SEO 标题、SEO 描述。

## 产品分类字段规则

产品分类导入支持两个字段：

- `Category Slugs`
- `Categories`

### 优先级

系统优先使用 `Category Slugs`，`Categories` 作为名称回退。

### 字段格式

多分类使用英文竖线 `|` 分隔。

`Category Slugs` 在产品 CSV 中默认只保存分类自身 slug，例如：

```text
phones|cables
```

`Categories` 可以保留名称层级路径，例如：

```text
电子产品 > 手机|电子产品 > 配件 > 线材
```

如果你需要单独维护分类层级，使用分类 CSV 中的 `Parent Slug` 字段。

导入时，`Category Slugs` 也兼容层级路径写法：

```text
electronics/phones|electronics/accessories/cables
```

### 导入行为

当处理每个分类路径时：

1. 如果存在 `Category Slugs`，按 slug 路径逐级查找分类。
2. 如果某一级不存在，则创建该级分类。
3. 如果缺少名称，会根据 slug 生成一个可读名称。
4. 如果没有 `Category Slugs`，则按 `Categories` 的名称路径逐级查找或创建。
5. 所有命中的分类最终都会绑定到当前产品。

这意味着：

- 产品导入即使没有单独上传分类 CSV，也可以在导入过程中自动补齐分类。
- 多层分类结构主要通过 `Categories` 和分类 CSV 的 `Parent Slug` 保留。

## 图片导入逻辑

图片 ZIP 为可选文件。

如果上传了 ZIP：

1. 解压到临时目录。
2. 建立图片索引。
3. 导入特色图和画廊图时，优先通过索引定位文件。
4. 找到文件后导入媒体库，并复用已存在的附件记录。

如果没有上传 ZIP：

- 不会阻止产品导入。
- 相关图片字段会记录日志，但不会创建附件。

## 日志与结果

导入过程中会写入文本日志，内容包括：

- 文件校验
- 分类创建或更新
- 产品创建或更新
- 图片处理结果
- 重试与异常信息

导入完成后：

- 成功结果会显示在后台通知中。
- 页面底部可以查看完整导入日志。
- 分类单独导入时，也会返回分类导入数量与错误数量。
